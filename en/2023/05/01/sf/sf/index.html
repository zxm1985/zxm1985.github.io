Ultra-free game development platform

introduce

The Super Free (SF Development) development platform is a cross-platform barrier-free game open community based on Lua language. By writing a set of codes, you can obtain the following cross-platform functions without knowing the underlying details of the code:
  • Handles user keyboard and touch input as well as mouse input.
  • Pack and encrypt sound effects for playback and mixing. Play 3D sound effects, attach effectors to your sound effects, etc.
  • Text to speech (TTS) local synthesis function.
  • Encapsulates functions such as network message passing and broadcasting, just like calling functions, and fully supports direct transmission of Lua tables.
  • Support online voice chat, and even support video interaction in the future, complete local control, self-developed.
  • Completely hide multi-threading issues on the server side, just like local single-threading logic.
  • Support custom storage of user data.
  • Supports 2D drawing, and will support 3D models in the future.

Currently the following platforms are supported: Windows7+, android8+, ios15.0+, mac osx11.11+, web (chrome, firefox)

Update log

The underlying functions of the platform will be continuously developed. For details, please click [Update Log](/2022/11/13/sf/update-log/)

Server script entry

ServerMain.lua View the ServerMain.lua script comments in detail. The server entry is written and placed in the FTP directory.
Assuming that the English name of the game is test, the ftp directory must be in )test-server/test/ServerMain.lua), which is the server entrance.
Currently, there is no need to log in to the WEB backend to restart the service. You can directly dissolve the room and reopen the room to use the new uploaded code.
Please log in to the WEB backend to view the server output log.

Client script entry

ClientMain.lua See lua comments for details
Assuming that the English name of the game is test, the ftp directory must be in )test-client/test/ClientMain.lua), which is the client update entrance.
Generally, the development package windows does not need to be updated, as long as the entry file is placed in the path of the source program, for example
resources/script/module/test/ClientMain.lua will take effect after restarting the client.
